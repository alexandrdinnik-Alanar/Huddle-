## Suggested project framing

Working project title:

Huddle Oslo MVP Validation — Technical and commercial validation of trusted family participation infrastructure

Core question:

Can Huddle’s adult-controlled family context, organiser data model and privacy-safe participation flow support a profitable and scalable local family infrastructure model?

Possible work packages:

1. MVP scope and technical risk clarification.
2. Family-context and recommendation logic validation.
3. Organiser data-quality and onboarding model.
4. Privacy and Trust & Safety architecture baseline.
5. Business-model and pricing validation.
6. Oslo pilot readiness plan.

## Budget fit

This route can support a more substantial validation project than Oppstartstilskudd 1, but budget rules and own-financing requirements must be checked before drafting.

Potential cost categories to verify:

- technical clarification;
- prototype/MVP validation;
- external expertise;
- business-model development;
- pilot-readiness work;
- privacy/security architecture where relevant.

Costs to treat carefully:

- generic marketing;
- completed product development;
- ordinary operations;
- costs started before application/approval;
- duplicate public support for the same cost base.

Innovation Norway requires disclosure of other public support, and the amount can be affected if other support covers the same project costs.
