# VIS-001 — Canonical design source reconciliation

## Task ID

`VIS-001`

## Objective

Establish one auditable design source of truth before base layouts or product pages are implemented.

## Dependency

`S0-014` is merged. `S0-015` must not start until this reconciliation is merged.

## Source precedence

1. accepted ADRs and repository security rules;
2. the approved Figma master file and its named canonical frames;
3. locked brand documentation and approved repository assets;
4. the canonical route map and merged application structure;
5. PNG screen sets as references where Figma has no approved frame;
6. archived or duplicate sources.

## Required outcomes

- identify the approved Figma file, pages, variables, styles, components, and product frames;
- classify visual sources as `PRODUCTION`, `REFERENCE`, `ARCHIVE`, `DUPLICATE`, or `UNDECIDED`;
- map every P0 route to its Figma coverage, repository reference sets, and React state;
- record approved non-P0 frames already present in the master file;
- make Figma-first approval mandatory for future visual work;
- leave the Figma file, application UI, and route implementation unchanged.

## Figma-first gate

For any new page, state, responsive variant, or material visual change:

1. create or update the design in the approved Figma master file;
2. use the approved brand variables, styles, components, and visual language;
3. obtain explicit visual approval;
4. add the approved node ID to the route registry;
5. only then implement it in React.

PNG screen sets may inform a proposal but cannot authorize production UI by themselves.

## Required files

- `docs/00-source-of-truth/design-source-of-truth.md`;
- `docs/01-product/design-source-inventory.csv`;
- `docs/01-product/screen-route-registry.csv`;
- `docs/execution/VIS-001/HANDOFF.md`;
- source-of-truth, roadmap, screen-corpus, and next-task pointers.

## Explicit exclusions

- no new Figma frames or edits;
- no page, layout, or component implementation;
- no deletion or movement of screen assets;
- no Code Connect mappings;
- no S0-015 or S0-016 implementation.

## Acceptance criteria

- the approved Figma master is named and linked by file key;
- confirmed product frames have stable node IDs;
- all P0 routes have an explicit design and implementation status;
- duplicate set 54 is recorded and cannot be mistaken for canonical output;
- missing Figma coverage is visible rather than inferred from PNG files;
- the next task is unambiguously S0-015, gated by Figma-approved layouts;
- documentation checks and the clean task diff pass.
