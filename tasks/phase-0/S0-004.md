## Objective

Implement the Huddle semantic design token foundation so future UI work uses approved brand-driven tokens instead of ad hoc values.

## Source reconciliation

Apply source precedence:

1. founder-approved visual identity source and reconciled brand audit;
2. merged repository implementation;
3. Sprint 0 Batch 2 source;
4. roadmap ordering and quality gates.

The token foundation must follow the approved Huddle brand direction from `Brand_Guidelines.zip` and `docs/00-source-of-truth/master-brand-system-audit.md`. Where older set50-derived values conflict, the approved master brand source wins.

Implementation rules:

- tokens must be semantic, not only raw scales;
- CSS variables must be generated or maintained in a structure that runtime styles can consume safely;
- tokens must be importable from application code without hardcoding duplicate brand constants elsewhere;
- deep green is the primary action color;
- copper is a brand accent, not the default CTA color everywhere;
- ivory or cream remains the base background direction;
- typography tokens must follow the approved Playfair Display and Inter direction from the brand audit;
- do not introduce dark mode or theming infrastructure in this task.
