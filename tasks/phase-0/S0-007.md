## 8. Required schema scope

Implement only the identity-base schema required by the roadmap and Sprint 0 source:

- enum `UserStatus`
- enum `RoleKey`
- model `User`
- model `Role`
- model `UserRole`
- the required relations between these models
- the required indexes for identity-base lookups
- explicit migration covering only `S0-007` scope

Do not extend beyond these identity-base structures in this task.

## 9. Required enums

Create these enums exactly:

### `UserStatus`

- `ACTIVE`
- `PENDING_VERIFICATION`
- `SUSPENDED`
- `DELETED`

### `RoleKey`

- `PARENT`
- `SELLER`
- `ORGANIZER`
- `EXPERT`
- `PARTNER`
- `MUNICIPALITY_USER`
- `MODERATOR`
- `ADMIN`
- `SUPER_ADMIN`
- `SYSTEM`

## 10. Required User model

`User` must include:

- `id` string primary key with `cuid` or `uuid` default
- `email` string unique
- `emailVerifiedAt` optional `DateTime`
- `name` optional string
- `avatarUrl` optional string
- `status` `UserStatus` with default `PENDING_VERIFICATION`
- `preferredLanguage` optional string
- `createdAt` `DateTime` default `now`
- `updatedAt` `DateTime` with `updatedAt`
- `deletedAt` optional `DateTime`
- relation to `UserRole`

## 11. Required Role model

`Role` must include:

- `id` string primary key with `cuid` or `uuid` default
- `key` `RoleKey` unique
- `name` string
- `description` optional string
- `createdAt` `DateTime` default `now`
- `updatedAt` `DateTime` with `updatedAt`
- relation to `UserRole`

## 12. Required UserRole model

`UserRole` must include:

- `id` string primary key with `cuid` or `uuid` default
- `userId` string
- `roleId` string
- `assignedById` optional string
- `assignedAt` `DateTime` default `now`
- `revokedAt` optional `DateTime`
- relation to `User`
- relation to `Role`
- optional `assignedBy` relation to `User`

## 13. Relation requirements

- `User` must support many role assignments through `UserRole`
- `Role` must support many assigned users through `UserRole`
- `UserRole` must join `User` and `Role`
- `UserRole.assignedById` must support optional attribution to a `User`
- relations must preserve historical role assignment records

## 14. Index requirements

Add indexes for:

- `User.status`
- `UserRole.userId`
- `UserRole.roleId`
- `UserRole.assignedById`

## 15. Multi-role requirements

- users must support multiple roles
- do not collapse roles into a single role field
- do not assume a user can hold only one role

## 16. Revocation-history requirements

- preserve revoked role assignment history using `revokedAt`
- do not delete role-assignment history merely because a role is revoked
- support auditing of who assigned a role where practical through `assignedById`

## 17. Migration requirements

- create an explicit, reviewable migration under `prisma/migrations/**`
- migration must represent only the `S0-007` identity-base schema scope
- no destructive migration
- no schema reset
- no production migration execution
- no production database connection
- no `db push` against production
- migration name and path must be documented in the handoff
