## 15. Input-shape requirements

Because `S0-009` does not resolve app-user roles:

- define a minimal explicit role-bearing input type for permission helpers
- it must clearly represent application role keys supplied by an already-resolved trusted caller
- it must not be `SupabaseAuthenticatedIdentity`
- it must not imply canonical app-user reconciliation happened in `S0-009`
- it should contain only what permission helpers need
- prefer a minimal shape such as:
  - `roles: readonly RoleKey[]`
- if an `id` field is included, it must be explicitly justified and must not be confused with provider identity

Trust-boundary rules:

- permission helpers are pure evaluators
- they must not fetch roles
- they must not trust arbitrary client-provided role strings
- callers are responsible for supplying validated application `RoleKey` values
- future app-user reconciliation and role loading remain outside `S0-010`
