# S0-017 — Create Sprint 0 QA checklist

## Task ID

`S0-017`

## Objective

Create an auditable Sprint 0 QA checklist that separates repository-verifiable evidence from external environment and operational prerequisites.

## Dependency

`S0-016` must be merged and its quality gates must pass.

## Required file

- `docs/sprint-0/qa-checklist.md`
- `docs/execution/S0-017/HANDOFF.md`

## Checklist coverage

1. Repository and toolchain
   - install, lint, typecheck, tests, build;
   - committed lockfile;
   - no committed secrets;
   - Node and pnpm versions.
2. Route architecture
   - public, auth, app, and admin groups;
   - 55-route placeholder manifest uniqueness;
   - dynamic route resolution;
   - correct base layout mapping.
3. Brand and design governance
   - approved brand assets and verification script;
   - semantic design tokens;
   - primitive UI components;
   - canonical Figma file and node registry;
   - no PNG or Claude prototype treated as production authority.
4. Database foundation
   - Prisma configuration;
   - migration history;
   - User, Role, UserRole, AuditLog models;
   - deterministic role seed;
   - clear distinction between static validation and execution against a real database.
5. Auth and authorization
   - Supabase client/session skeleton;
   - external identity versus canonical app-user boundary;
   - permission helpers and guard helpers;
   - no client-side visibility used as authorization;
   - deferred app-user reconciliation and role loading are explicit.
6. API foundation
   - response envelope and error codes;
   - input validation boundary;
   - privacy-safe error output;
   - audit helper behavior.
7. Scope control
   - no production marketplace, activities, events, community, messaging, payments, municipality, or admin operations implemented during Sprint 0.
8. External prerequisites
   - development Supabase project;
   - safe development database;
   - Vercel project;
   - staging/production separation;
   - monitoring, email, and other provider decisions.
9. Rejection criteria
   - missing handoff;
   - failing quality gate;
   - missing task contract;
   - unapproved Figma-dependent visual decision;
   - invented authorization input;
   - secrets or sensitive data in repository/logs.
10. Sprint 1 readiness gate
   - evidence for each completed Sprint 0 task;
   - explicit list of remaining external blockers;
   - founder/product decisions that cannot be inferred by agents.

## Requirements

- Use pass/fail/not-applicable/not-verifiable status fields.
- Include an evidence column or evidence location for every check.
- Do not mark external services complete from repository code alone.
- Do not treat a placeholder route as a completed product flow.
- Do not treat approved design as implemented UI.
- Link to `launch-readiness.md` for later launch gates.

## Acceptance criteria

- the checklist is detailed enough for independent manual review;
- repository evidence and external evidence are clearly separated;
- rejection criteria are explicit;
- Sprint 1 readiness is not declared while required evidence is missing;
- required quality gates pass;
- `docs/execution/S0-017/HANDOFF.md` is complete.

## Quality gates

- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm format:check`
- `git diff --check`

## Stop condition

Stop after S0-017. Do not begin S0-018 in the same branch.
