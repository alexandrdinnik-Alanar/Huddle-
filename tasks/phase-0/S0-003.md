## Unblock procedure

1. Supply all exact approved binary assets, or explicitly approve a smaller canonical subset and update the task contract through review.
2. Preserve original bytes and filenames on initial import.
3. Calculate SHA-256 for every imported binary.
4. Replace expected-only entries in `public/brand/expected-assets.json` with verified checksum records.
5. Perform visual owner review against the approved source.
6. Only then implement components.

## Component contract after unblock

Create:

```text
src/components/huddle/BrandLogo.tsx
src/components/huddle/IconWatermark.tsx
```

`BrandLogo` must:

- render approved supplied logo assets only;
- support only manifest-backed variants;
- support meaningful alt text;
- support dimensions and/or `className`;
- never render a typed text logo;
- never recolor assets with CSS filters.

`IconWatermark` must:

- render approved supplied icon assets only;
- support only manifest-backed variants;
- default to decorative `aria-hidden` behavior;
- allow watermark/background/seal use;
- not invent a new lockup.

## Explicitly forbidden

- crop from `510_logo_usage_brand_rules.png`;
- redraw or trace geometry;
- AI-generate a replacement;
- type `Huddle` as a logo substitute;
- create SVG recreations from screenshots;
- recolor with CSS `filter`;
- invent missing variants;
- silently use another brand's asset.
